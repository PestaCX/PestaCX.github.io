    Mire jó a JavaScript?

    A weboldalakat a html-kód többnyire csak statikus, vagyis nem változó adatokkal tudja feltölteni. Ennek a következménye, hogy a weboldal "buta" marad, hiszen egy egyszerű számtani műveletet sem tud elvégezni. Ezen segít a JavaScript (a továbbiakban csak JS), mivel a JS képes az adatokkal bánni. A megjelenítés továbbra is a HTML kód feladata lesz.


    Hogyan kapcsolódik a JS a HTML-hez?

    Ennek két módja lehetséges:

    • inline - a html-kód sorai között
    • external - ekkor egy külső fájlban van a script, és csak arra hivatkozunk a html-ben

    A JS programoknak a neve "script" - mint ahogy a JavaScript nevében benne is van. Egy komplexebb weboldalon nagyon sok script futhat attól függően, hogy mennyi olyan elem van a weblapon, amelyik programozást, számítást igényel.
    Egyértelmű, hogy ha egy weblapon közvetlen a HTML-kódban adjuk meg a JS scripteket, akkor a html-kódunk egyre bonyolultabb, áttekinthetetlenebb lesz - hasonlóan, mint ahogy a CSS is összezavarja a kódot, ha túl sok van belőle.

    A JS script(ek)et a <script>...</script> tag-ek által határolt szekcióba lehet írni. Egy html-fájlban több helyen is lehet script.

    A megadás módja inline-kódként:

    <body>
      <script>
         ... JavaScript program
      </script>
    </body>

    Vagy a külső JS-et tartalmazó fájlra való hivatkozás:

      <script href="JS_fájl">
    Vagy inkább így:
      <script>> href="JS_fájl"</script>

    /A JS-fájlt természetesen elérési útvonallal együtt kell megadni!


    Bár JS lehet elvileg a head vagy a body szekcióban is, legcélszerűbb a body végére, tehát a html értelmezése utáni részre tenni, hogy a JS számára rendelkezésre álljon a html-ben megadott elem. (Mivelhogy a html-fájl értelmezése fentről lefelé történik.) Fordított esetben a JS-et olyan művelet elvégzésére kényszeríthetjük, amihez még nem érhető el a szükséges html-elem.
